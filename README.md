# 🏠 邻里APP (NearUs)

一个现代化的社区邻里互动平台，基于React + Flask开发，采用2025年最新的UI/UX设计标准。

## ✨ 主要功能

- 🗨️ **邻里动态** - 发布和浏览社区动态，分享生活点滴
- 👥 **社区群组** - 创建和加入兴趣群组，结识邻居朋友  
- 🎯 **任务中心** - 发布和接受邻里互助任务
- 🎉 **活动广场** - 组织和参与社区活动
- 🏆 **积分排行** - 积分系统激励社区参与
- 👤 **个人中心** - 管理个人信息和查看参与记录

## 🚀 启动指南

### 环境要求
- Python 3.7+
- Node.js 16+
- npm 6+

### 🎯 一键启动（推荐）

#### Windows用户
```bash
# 双击运行批处理文件
start_app.bat
```

#### 创建演示数据
```bash
# 双击运行演示数据生成脚本
create_demo.bat
```

**注意**：首次使用建议先运行演示数据生成脚本，然后再启动应用。

### 📋 脚本文件说明

| 脚本文件 | 功能 | 使用场景 |
|---------|------|----------|
| `start_app.bat` | 一键启动前后端服务 | 日常启动应用 |
| `stop_app.bat` | 停止所有服务 | 正常关闭应用 |
| `force_stop.bat` | 强制停止所有进程 | 服务无法正常停止时 |
| `create_demo.bat` | 创建演示数据 | 首次使用或重置数据 |
| `fix_venv.bat` | 修复虚拟环境 | 依赖安装失败时 |

### 🔧 脚本特性

- **英文界面**：所有脚本使用英文输出，避免编码问题
- **自动检测**：自动检查Python、Node.js环境
- **依赖管理**：自动安装Python和前端依赖
- **错误处理**：详细的错误提示和故障排除

### 🛠️ 故障排除

如果脚本运行出现问题：

1. **依赖安装失败**：
   - 运行 `fix_venv.bat` 修复虚拟环境
   - 这会删除并重新创建虚拟环境，解决包元数据损坏问题

2. **环境检查**：
   - 确保Python和Node.js已安装并在PATH中
   - 运行 `python --version` 和 `node --version` 验证

3. **端口占用**：
   - 检查端口3000和5000是否被占用
   - 使用 `netstat -an | findstr :3000` 检查

4. **权限问题**：
   - 尝试以管理员身份运行脚本
   - 确保防火墙允许本地端口访问

5. **网络问题**：
   - 检查网络连接
   - 如果在中国大陆，可能需要配置pip镜像源

#### 手动启动步骤

#### 1. 克隆项目
```bash
git clone https://github.com/happysheep321/NearUs.git
cd NearUs
```

#### 2. 启动后端服务
```bash
# 创建虚拟环境
python -m venv venv

# 激活虚拟环境 (Windows)
venv\Scripts\activate.bat
# 激活虚拟环境 (Linux/Mac)
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 启动后端服务
python app.py
```

#### 3. 启动前端服务
```bash
# 新开一个终端窗口
cd frontend

# 安装依赖
npm install

# 启动前端服务
npm start
```

#### 4. 访问应用
- 前端地址：http://localhost:3000
- 后端地址：http://localhost:5000

### 停止服务
- **推荐方法**：在各自的服务窗口中按 `Ctrl+C` 停止
- **一键停止**：使用 `stop_app.bat` 停止所有相关进程
- **强制停止**：如果普通停止无效，使用 `force_stop.bat` 强制关闭

**注意**：如果停止后页面仍能访问，说明进程没有完全关闭，请使用强制停止脚本。
